# Mindlex 주요 기능 명세서

이 문서는 Mindlex(마인드 용어 사전/퀴즈/단어장) 서비스의 핵심 기능 및 비기능 요구사항을 정리한 명세서입니다.

---

## 1. 용어 사전 기능

| 코드   | 설명                                                                 |
|--------|----------------------------------------------------------------------|
| F-1.1  | 용어 목록 페이지를 제공한다. (A-Z 또는 가나다 순 정렬)                |
| F-1.2  | 각 용어 클릭 시 상세 페이지로 이동하며, 정의, 예시, 관련 법률, 태그를 보여준다. |
| F-1.3  | 용어에는 태그를 붙여 주제별 분류(예: 정신질환, 실천기술, 법제도 등)가 가능하다. |
| F-1.4  | 검색 기능을 통해 용어를 키워드 기반으로 필터링할 수 있다.            |

## 2. 퀴즈 학습 기능

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-2.1  | 랜덤 또는 주제별로 퀴즈를 제공한다.                          |
| F-2.2  | OX 퀴즈, 객관식 퀴즈 형식 중 선택 가능하다.                  |
| F-2.3  | 퀴즈 결과를 즉시 피드백하고 정답 설명을 제공한다.            |

## 3. 개인 단어장 기능

| 코드   | 설명                                                                 |
|--------|----------------------------------------------------------------------|
| F-3.1  | 사용자가 좋아하는 단어를 "내 단어장"에 추가할 수 있다.              |
| F-3.2  | 로그인 없이 localStorage 기반 저장이 가능하며, 추후 로그인 기반 확장도 고려 |
| F-3.3  | 단어장 내에서도 복습 및 퀴즈 기능이 가능하다.                        |

## 4. 검색 및 필터링 기능

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-4.1  | 전체 단어를 키워드로 검색할 수 있다.                         |
| F-4.2  | 태그별 필터링 가능 (예: 중독, 아동, 재활, 법제도 등)          |
| F-4.3  | 단어의 가나다순, 인기순 등 정렬 기능을 제공한다.              |

## 5. TTS(음성 지원) 기능 (선택 사항)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-5.1  | 버튼 클릭 시 해당 용어의 발음이 TTS로 재생된다.               |
| F-5.2  | 복잡한 용어나 영어 용어에 유용함                             |

## 6. 사용자 참여/커뮤니티 기능 (확장 제안)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-6.1  | 용어/퀴즈/예시 등에 대한 사용자 댓글, 의견, 평가(좋아요/싫어요) 기능 |
| F-6.2  | 사용자 제안 용어 등록 및 관리자 승인 프로세스                |
| F-6.3  | 오답노트/단어장 공유(링크, 소셜 등)                         |

## 7. 개인화/통계 기능 (확장 제안)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-7.1  | 학습/퀴즈 이력, 진도, 정답률 등 개인별 통계 대시보드         |
| F-7.2  | 맞춤 퀴즈 추천(자주 틀린 문제, 관심 태그 기반)               |
| F-7.3  | 최근 본 용어/퀴즈 자동 저장 및 빠른 접근                     |

## 8. 알림/리마인더 (확장 제안)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-8.1  | 퀴즈/복습 리마인더(웹 푸시, 이메일 등)                      |
| F-8.2  | 신규 용어/퀴즈/이벤트 알림                                   |

## 9. 접근성/다국어 (확장 제안)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-9.1  | 다국어(영어, 중국어 등) 지원 및 번역                         |
| F-9.2  | 고대비 모드, 폰트 크기 조절 등 접근성 강화                  |

## 10. 관리자/운영 기능 (확장 제안)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-10.1 | 관리자 대시보드(용어/퀴즈/사용자 관리, 통계 시각화)          |
| F-10.2 | 신고/피드백 관리, 데이터 백업/복원                           |

## 11. 외부 연동/확장 (확장 제안)

| 코드   | 설명                                                         |
|--------|--------------------------------------------------------------|
| F-11.1 | 외부 API(공공데이터, 위키 등)와 연동한 용어 자동 업데이트    |
| F-11.2 | 모바일 앱(PWA) 지원                                         |

---

## 비기능 요구사항

| 코드   | 설명                                                                 |
|--------|----------------------------------------------------------------------|
| NF-1   | 반응형 UI로 모바일, 태블릿, 데스크탑 모두 최적화                   |
| NF-2   | 빠른 페이지 로딩을 위한 정적 생성(SSG) 우선 사용                   |
| NF-3   | 접근성과 UX 향상을 위한 시멘틱 마크업 및 ARIA 속성 고려            |
| NF-4   | 사이트는 Next.js (App Router) 기반으로 제작                        |
| NF-5   | 배포는 Vercel 등을 통해 안정적인 퍼블릭 액세스 보장                |
